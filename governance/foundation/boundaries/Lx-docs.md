# Lx — Documentation Boundary

## Boundary role

Lx defines separation between:
- normative law surfaces (binding)
- explanatory/documentary surfaces (non-binding)

## Normative surfaces

Binding sources:
- `foundation/*`
- normative contracts under `contracts/*`
- formal artifacts where explicitly normative

## Explanatory surfaces

Informative docs may explain but cannot override normative content.

## Ontology alignment

Documentation must use primary ontology (`core/exec/brain` + `protocol/platform/support`).
Historical terms may appear only as migration aliases.

## Drift rule

If docs diverge from normative law, docs are wrong and must be corrected.
