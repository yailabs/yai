# Economic

`economic` is a high-sensitivity control family in the six-layer canonical stack.

## Scope

This family governs operations that affect value movement, financial state, counterparty exposure,
and financial reporting integrity.

## Primary specialization set

- `payments`
- `transfers`
- `settlements`
- `fraud-risk-controls`
- `treasury`
- `accounting-integrity`
- `credit-exposure`
- `invoicing`
- `pricing-and-quoting`

## Normative profile

- Strong authority burden for high-impact actions
- Default review/escalation under sector-finance and regulatory overlays
- Evidence hardening for approval chain and provenance
- Fail-closed posture for non-contracted or non-approved value movements

## Overlay relevance

- Regulatory: `gdpr-eu`, `retention-governance`, `security-supply-chain`
- Sector: `sector.finance`
- Contextual: `organization`, `workspace`, `session`

Legacy seed payload remains in `transitional/domain-family-seed/` for bridge compatibility.
