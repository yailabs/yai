# L1 — Runtime Core Sovereignty Boundary

## Boundary role

L1 is the sovereign boundary of `runtime-core`.
It defines where authority, lifecycle control, workspace sovereignty, and enforcement decisions become binding.

## Primary semantics

- authority evaluation and decision legitimacy
- lifecycle gating and workspace sovereignty
- dispatch/control-plane admissibility
- non-bypassable enforcement entry

## Ontology alignment

Primary term: `runtime-core`.
Historical label `kernel` is retained only as migration alias.

## Prohibited patterns

- authority decisions outside L1
- execution/effect paths bypassing L1 governance
