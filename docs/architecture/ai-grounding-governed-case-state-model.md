# AI Grounding over Governed Case State Model (QG-3)

## Purpose

Define the canonical grounding model so agents and `exec` reason over owner-side
governed case state, not over unscoped raw runtime exhaust.

## Canonical thesis

AI reasons over governed case state, not over unscoped raw runtime exhaust.

Grounded AI is context-disciplined intelligence, not sovereign adjudication.

## Context stack model

QG-3 locks four context layers:

1. raw input material (events/log fragments/runtime signals)
2. structured inspect/query surfaces (QG-1)
3. unified governed graph representation (QG-2)
4. task-scoped AI grounding context (QG-3 projection)

Agents consume layer 4, derived from layers 2+3 under owner governance.

## Governed grounding substrate

Grounding context may include:

- canonical workspace/case anchors
- source/edge/mesh runtime summaries
- binding/action-point/delegation summaries
- grant/snapshot/envelope validity/freshness state
- ingress/transport/overlay condition where relevant
- conflict/hold/escalation context
- provenance and adjudication-boundary markers

## Exec positioning

`exec` is the disciplined consumer/orchestrator of governed grounding context:

- builds task-scoped grounding projection
- keeps policy/authority boundaries explicit
- avoids direct reasoning over blind raw exhaust
- routes proposals/suggestions through owner-governed paths

`exec` does not become canonical truth authority.

## Scope and boundary rules

- grounding context is task-scoped and relevance-filtered
- context remains freshness-aware and validity-aware
- context remains provenance-aware and auditable
- AI output is advisory/execution-support unless separately adjudicated
- final authority and canonical truth remain owner-side

## Auditability and provenance

Grounding model must preserve:

- workspace/context identity used for reasoning
- summary/graph state shape at reasoning time
- adjudication boundary markers (`observed|accepted|canonicalized`)
- degradation/unavailability reasons when grounding is partial

This enables explainable and reviewable AI-assisted operation.

## Distributed-state handling

Peer/transport/mesh state can be included in grounding only after owner-side
recomposition and governance checks. Peer-local claims are not blindly promoted
to grounding truth.

## Handoffs

QG-3 provides the base for:

- DX-2 inspect surfaces showing what AI is grounded on
- DX-3 governance alignment for policy/trust/provenance constraints
- QW-6 readiness evidence for governed AI operation
