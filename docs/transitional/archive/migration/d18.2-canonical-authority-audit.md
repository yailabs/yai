# D18.2 Canonical Authority Audit

## Scope
Authority audit of live documentation (`docs/**` excluding `docs/archive/**`) to remove implicit ownership and close remaining source-of-truth ambiguity.

## Authority Conflict Clusters
| Cluster | Competing Surfaces | Canonical Owner | Resolution |
| --- | --- | --- | --- |
| Runtime model and execution boundaries | runtime architecture notes, runbook operational details | `docs/architecture/runtime/architecture.md` | Runbooks explicitly marked `procedural` and linked through `depends_on`. |
| Workspace semantics and lifecycle | workspace architecture satellites and workflow guides | `docs/architecture/workspace/architecture.md` | Guides remain `support`; architecture stays canonical. |
| Distributed runtime behavior | peering/policy docs vs operational evidence notes | `docs/architecture/mesh/architecture.md` | Program evidence constrained to reports; architecture remains model authority. |
| Protocol semantics and lookup | protocol architecture vs reference family readmes | `docs/reference/protocol/README.md` (lookup), `docs/architecture/protocol/transport.md` (model) | Cross-section precedence documented; reference and architecture separated by role. |
| Data and intelligence runtime | architecture satellites and old implementation notes | `docs/architecture/data/architecture.md`, `docs/architecture/edge/daemon-edge-architecture.md` | Canonical owners declared via `primary_for`; support docs de-emphasized. |
| Program evidence vs system authority | reports vs architecture/reference | `docs/program/reports/runtime-convergence-report.md` and `docs/program/reports/audit-convergence-report.md` as evidence only | Reports marked `support`; precedence model blocks them from overriding architecture/reference. |

## Live Role Distribution (High Visibility)
- `canonical`: architecture owner docs.
- `reference`: protocol/model/schema/model/registry/command/cli/sdk owners.
- `procedural`: core runbooks.
- `support`: section spines, guides, program reports.
- `historical`: restricted to archive-only scope.

## Declassification and Conflict Outcome
- Competing ownership in live docs removed for primary topics listed in D18.2 source map.
- Historical material is non-authoritative by contract and by validator checks.
- All high-visibility docs now expose role/status/owner metadata, with dependency links where required.
